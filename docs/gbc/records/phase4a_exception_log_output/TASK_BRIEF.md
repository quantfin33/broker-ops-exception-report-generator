# GBC Task Brief - Phase 4A Exception Log Output

## Goal

Generate `outputs/order_exception_log.csv` from validated static inputs and existing exception records.

## Background

Phase 3C through Phase 3E created internal exception type, severity, and recommended-action metadata. Phase 4A is the first output phase and writes only the order exception log CSV.

## In Scope

- Add a standard-library CSV report writer.
- Implement `generate-reports --report exception-log`.
- Validate inputs before writing.
- Write only `outputs/order_exception_log.csv`.
- Add tests for command behavior, headers, row count, severity counts, recommended actions, and absence of other outputs.
- Update Phase 4A docs, audit ledger, project tracker, and project-plan PDF.

## Out Of Scope

- JSON shift summary.
- By-symbol statistics.
- Markdown shift report.
- Excel or HTML output.
- Market-event overlap logic.
- Duplicate-ID exception reporting.
- Missing-field exception reporting.
- New exception types.
- Severity or recommended-action changes.
- External dependencies.
- Live APIs, MT4/MT5, FIX, broker bridge, Binance, Telegram, TradingView, account, credential, `.env`, or browser-session dependencies.
- README edits.
- Commit.

## Acceptance Criteria

- [x] `generate-reports --report exception-log` exits 0 for canonical fixtures.
- [x] `order_exception_log.csv` has the approved headers.
- [x] `order_exception_log.csv` has 14 data rows.
- [x] Severity counts are `Critical: 4`, `Warning: 10`, `Info: 0`.
- [x] Recommended-action coverage is 14/14.
- [x] No JSON, by-symbol, Markdown, Excel, or HTML report output is generated by Phase 4A.

## Safety Check

- [x] Static/demo-only.
- [x] No live trading, broker API, MT4/MT5, exchange, Telegram, or account dependency.
- [x] No credentials, browser sessions, app-support files, or private account data.
